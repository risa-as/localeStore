# Handoff: لوحة تحكم تحليل الأرباح (Profit Analytics Dashboard)

## Overview
صفحة تحليل أرباح لمتجر إلكتروني عراقي (العملة: دينار عراقي `د.ع`، الاتجاه RTL، اللغة عربية).
تعرض ملخّص الإيرادات والتكاليف، المصاريف القابلة للإدخال، تطوّر الربح عبر الفترة،
توزيع الإيراد على المنتجات، وجدول تفصيلي للمنتجات مرتّب حسب الربح.
الهدف: أن يفهم صاحب المتجر ربحه الحقيقي بعد كل التكاليف بلمحة واحدة.

## About the Design Files
الملفات المرفقة **مرجع تصميمي مكتوب بـ HTML** — نموذج أوّلي يوضّح الشكل والسلوك المطلوبين،
وليس كوداً جاهزاً للنسخ إلى الإنتاج. المطلوب **إعادة بناء هذا التصميم في بيئة المشروع الحالية**:
**Next.js (App Router) + TypeScript + Tailwind CSS + shadcn/ui**، باستخدام أنماط ومكوّنات المشروع.
- الملف `لوحة تحكم تحليل الأرباح.dc.html` هو نموذج تفاعلي كامل (منطق الحساب داخل كلاس JS) — استخدمه كمصدر الحقيقة للأرقام والصِيَغ.
- تجاهل بنية الـ "Design Component" (`x-dc`, `DCLogic`, `support.js`) — هي خاصة ببيئة التصميم فقط. أعد كتابة المنطق كـ React hooks/components عادية.

## Fidelity
**High-fidelity (hifi)** — ألوان وطباعة ومسافات وتفاعلات نهائية. أعد بناء الواجهة بدقّة بصرية عالية
مستخدماً مكوّنات shadcn/ui (Card, Table, Input, Button, Tabs, Badge) وTailwind، مع الحفاظ على الألوان أدناه بدقّة.

## Global Setup
- **Direction**: `dir="rtl"` `lang="ar"` على الحاوية الجذر.
- **Font**: `Tajawal` (Google Fonts, أوزان 400/500/700/800/900). استخدم `next/font/google`.
- **Theme**: وضعان Light/Dark عبر مفتاح تبديل في الهيدر. استخدم `next-themes` + متغيّرات CSS (`class` strategy في Tailwind). الافتراضي: Dark.
- **Currency formatter**: `new Intl.NumberFormat('ar-IQ').format(Math.round(n)) + ' د.ع'`.
- **Layout**: صفحة مستقلة (بدون sidebar). `max-width: 1440px` موسّطة، padding أفقي 28px.
  هيدر لاصق أعلى (`sticky top-0`)، وشريط ملخّص لاصق أسفل (`sticky bottom-0`).

## Screens / Views — one page, top to bottom

**1. Header (sticky, backdrop-blur)**
- يمين: `تحليل الأرباح` (h1) + pill أخضر `{orders} طلب مكتمل`؛ تحته سطر وصف رمادي.
- يسار: صندوق فترة (من/إلى مع فاصل عمودي) — قِيَم `07/01/2026` → `07/21/2026`؛
  زر ثانوي `⎙ PDF` (يستدعي `window.print()`)؛ زر أساسي أخضر `⭳ تصدير Excel` (يصدّر CSV)؛ مفتاح تبديل الثيم (toggle 52×30، مقبض 24px).

**2. Insights row — 3 بطاقات** (نقطة ملوّنة + label/value/meta):
- أعلى منتج ربحاً (نقطة خضراء) — الاسم + `{fmt} صافي ربح`.
- أقل هامش ربح (نقطة حمراء) — الاسم + `هامش {x}%`.
- متوسط الربح لكل طلب (نقطة زرقاء) — `netProfit / orders`.

**3. ملخّص الإيرادات والتكاليف — 4 بطاقات KPI** ثم **شريط تركيب التكلفة**:
- بطاقات: `إجمالي ما حُصِّل` 1,034,000 · `تكلفة التوصيل الفعلية` 113,000 (meta: بغداد 28,000 / غيرها 85,000) · `تكلفة البضاعة` 453,500 (24 قطعة) · `صافي الربح` 467,500 (مميّزة: إطار أخضر + قيمة خضراء + emphBg، هامش 50.8%).
- شريط أفقي مقسّم بنِسَب من "ما حُصِّل": تكلفة البضاعة / تكلفة التوصيل / المصاريف / صافي الربح، مع مفتاح تحته (نقطة + label + %).

**4. المصاريف — 4 بطاقات (بطاقتان بإدخال حي):**
- `مصاريف الإعلانات`: `<input type=number>` كبير بلا حدود، يحدّث الحالة.
- `المصاريف التشغيلية`: نفس الشيء.
- `إجمالي المصاريف` = ads + opex (محسوبة).
- `صافي الربح الحقيقي` (مميّزة، خضراء) = `467500 − allExpenses`؛ meta هامش حقيقي = `realProfit / 921000 * 100`.
- عند `adsExpense > 0`: توزّع على المنتجات حسب حصة كل منتج من الإيراد (`sharePct/100`) وتظهر أعمدة إعلان في الجدول (انظر أدناه).

**5. تنبيه هامش منخفض** (يظهر شرطياً إن وُجد منتج هامشه < 15%):
- بطاقة حمراء ناعمة، أيقونة `!` دائرية حمراء + عنوان، ثم صفوف (اسم المنتج + `هامش {x}%`).
- حالياً يظهر: `سيت فاتح مفاصل` (هامش 6.1%).

**6. رسم بياني (Card) — تبويبات:**
- 3 تبويبات: `صافي الربح` (خط تراكمي + تعبئة تدرّج) · `الإيراد` (أعمدة يومية) · `الطلبات` (أعمدة يومية).
- SVG viewBox `0 0 800 260`، 4 خطوط شبكة أفقية. الخط/الأعمدة باللون الأخضر. 6 تسميات محور س: `01/07,05/07,09/07,13/07,17/07,21/07`.
- أعلى يمين البطاقة: 4 شارات مقارنة بالفترة السابقة (▲/▼ + % أو نقاط + "مقابل الفترة السابقة") لِـ: ما حُصِّل، صافي الربح الحقيقي، هامش الربح (نقاط)، عدد الطلبات. الأخضر للارتفاع، الأحمر للانخفاض.
- بيانات المقارنة (الفترة السابقة): collected 890,000 · realProfit 398,000 · realMargin 44.7% · orders 19.

**7. توزيع الطلبات والتوصيل — 3 بطاقات:** بغداد (7 طلب، شريط تقدّم أخضر، 28,000) · باقي المحافظات (17 طلب، شريط رمادي، 85,000) · إجمالي التوصيل (113,000). النِسَب من 24.

**8. توزيع الإيراد على المنتجات — donut:**
- دائرة `conic-gradient` بألوان donutPalette حسب `sharePct` لكل منتج، مع "ثقب" مركزي (بلون البطاقة) يعرض إجمالي الإيراد الصافي 921,000.
- مفتاح شبكي: نقطة ملوّنة + اسم المنتج + `{fmt} · {pct}%`.
- (في React استخدم مكتبة مثل recharts PieChart أو ابقِ الـ conic-gradient كما هو.)

**9. تفصيل المنتجات — جدول:**
- شريط علوي: `<input>` بحث بالاسم + عدّاد `{n} من {total} منتج`.
- أعمدة: # (شارة رتبة؛ ميدالية ذهب/فضة/برونز لأول 3) · المنتج · الكمية · الراجع · الإيراد الصافي · تكلفة البضاعة · صافي الربح (أخضر) · هامش الربح (badge أخضر/أحمر) · % من الإيراد (شريط صغير + نسبة).
- **فرز**: رؤوس المنتج/الكمية/الإيراد/الربح/الهامش قابلة للنقر (تبديل asc/desc، سهم ↑/↓).
- **صف قابل للتوسيع**: النقر على أي صف يفتح تفاصيل اقتصاديات الوحدة: سعر بيع الوحدة (صافي)، تكلفة الوحدة، ربح الوحدة (أخضر)، القطع المباعة، الراجع + ملاحظة FIFO.
- **أعمدة الإعلان الشرطية** (عند adsExpense>0): مصاريف الإعلان، إعلان/طلب، الربح بعد الإعلان، هامش بعد الإعلان.
- **tfoot المجموع**: 24 · راجع 1 (4.0%) · إيراد 921,000 · تكلفة 453,500 · ربح 467,500 (أخضر) · هامش 50.8% · 100%.

**10. شريط ملخّص لاصق أسفل (sticky):** إجمالي ما حُصِّل · إجمالي المصاريف (توصيل+بضاعة+تشغيل+إعلان) · صافي الربح الحقيقي (أخضر) · هامش الربح الحقيقي (أخضر).

## State Management
- `theme: 'dark' | 'light'` — مفتاح الهيدر.
- `adsExpense: number`, `opexExpense: number` — إدخال حي؛ يعيد حساب صافي الربح الحقيقي، الهامش، شريط التركيب، الشريط السفلي، وأعمدة الإعلان.
- `searchQuery: string` — تصفية جدول المنتجات بالاسم.
- `sortKey: 'name'|'qty'|'revenue'|'profit'|'margin' | null`, `sortDir: 'asc'|'desc'`.
- `chartMetric: 'profit'|'revenue'|'orders'` — تبويب الرسم.
- `expandedRank: number | null` — الصف الموسّع.
- لا يوجد جلب بيانات؛ الأرقام ثابتة (mock). في الإنتاج استبدلها بـ API فعلي.

## Interactions & Behavior
- تبديل الثيم فوري (transition 0.25s على الخلفية).
- إدخال المصاريف `onChange` (فارغ = 0)؛ كل المشتقّات تُعاد حسابها.
- الفرز: نقرة تبدّل الاتجاه؛ الأسماء `localeCompare(…, 'ar')`.
- توسيع الصف: نقرة تفتح/تغلق؛ سهم ▸/▾ يمين اسم المنتج؛ خلفية الصف الموسّع خضراء ناعمة.
- تصدير Excel: يبني CSV بـ BOM (`\uFEFF`) لدعم العربية ويُنزّله.
- PDF: `window.print()`؛ عناصر `.no-print` (الهيدر، الشريط السفلي) مخفيّة عند الطباعة.
- كل الجداول العريضة داخل حاوية `overflow-x:auto`.
